# コンポーネント詳細分析

## エージェント詳細

### 1. planner（計画エージェント）
**目的**: 複雑な機能実装前の計画立案

**主な機能**:
- 要件の再定義と明確化
- フェーズ分けした実装ステップ
- リスク評価と依存関係の特定
- ユーザー承認待ちの確認フロー

**利用シーン**: 新機能追加、大規模リファクタリング

---

### 2. tdd-guide（TDDガイドエージェント）
**目的**: テスト駆動開発の強制

**主な機能**:
- インターフェース定義（スキャフォルド）
- 失敗するテスト作成（RED）
- 最小実装（GREEN）
- リファクタリング
- 80%カバレッジ確認

**利用シーン**: 新機能実装、バグ修正、コアロジック開発

---

### 3. security-reviewer（セキュリティレビューエージェント）
**目的**: 脆弱性の検出と修正

**チェック項目**:
- OWASP Top 10
- ハードコードされたシークレット
- SQLインジェクション
- XSS
- 入力バリデーション
- 認証/認可

**利用シーン**: コミット前、API開発、認証機能実装

---

### 4. build-error-resolver（ビルドエラー解決エージェント）
**目的**: ビルドエラーの迅速な修正

**戦略**:
- 最小限の差分で修正
- アーキテクチャ変更を避ける
- 一度に1エラーずつ対処
- 修正後の検証

**利用シーン**: ビルド失敗時、TypeScriptエラー

---

### 5. e2e-runner（E2Eテストエージェント）
**目的**: Playwrightを使用したE2Eテスト管理

**機能**:
- Page Object Modelパターン
- 不安定なテストの検出と隔離
- スクリーンショット/動画/トレース
- CI/CD統合

**利用シーン**: 重要なユーザーフロー、リリース前テスト

---
