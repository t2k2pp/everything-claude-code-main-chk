# リファクタリングクリーン

テスト検証で不要コードを安全に特定して削除:

1. 不要コード分析ツールを実行:
   - knip: 未使用のエクスポートとファイルを見つける
   - depcheck: 未使用の依存関係を見つける
   - ts-prune: 未使用のTypeScriptエクスポートを見つける

2. .reports/dead-code-analysis.md に包括的なレポートを生成

3. 発見を重大度で分類:
   - 安全: テストファイル、未使用のユーティリティ
   - 注意: APIルート、コンポーネント
   - 危険: 設定ファイル、メインエントリポイント

4. 安全な削除のみを提案

5. 各削除前に:
   - 完全なテストスイートを実行
   - テストが成功することを確認
   - 変更を適用
   - テストを再実行
   - テストが失敗したらロールバック

6. クリーンしたアイテムのサマリーを表示

テストを実行せずにコードを削除しないでください！
